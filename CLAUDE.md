# 가족 건강검진 관리 앱 - 프로젝트 설계

## 프로젝트 개요
가족 구성원의 건강검진 결과를 저장·관리하는 개인용 웹앱.
매년 검진 시 이전 기록을 쉽게 확인하고, 병원 방문 시 검사 이력을 즉시 조회할 수 있게 한다.

---

## 기술 스택

| 영역 | 기술 |
|---|---|
| Frontend | Next.js 14 (App Router) + TypeScript + Tailwind CSS |
| Backend/DB | Supabase (PostgreSQL + Auth + Storage) |
| 배포 | Vercel (프론트) + Supabase (백엔드) |
| 플랫폼 | 웹앱 (모바일 브라우저 최적화 PWA) |

## 데이터 모델

```sql
-- 가족 그룹
families (
  id uuid PK,
  name text,              -- "홍길동 가족"
  invite_code text,       -- 초대 링크용 코드
  created_at timestamp
)

-- 가족 구성원
members (
  id uuid PK,
  family_id uuid FK,
  name text,
  birth_date date,
  gender text,            -- 'M' | 'F'
  created_at timestamp
)

-- 검진 기록 (건강검진 or 단일검사 모두)
checkup_records (
  id uuid PK,
  member_id uuid FK,
  type text,              -- 'checkup' (건강검진) | 'single' (단일검사)
  record_date date,
  hospital text,
  notes text,
  created_at timestamp
)

-- 항목별 결과 수치
checkup_items (
  id uuid PK,
  record_id uuid FK,
  item_code text,         -- 표준 항목 코드 (예: 'blood_pressure_sys')
  item_name text,         -- 표시 이름 (예: '수축기혈압')
  value text,             -- 수치 or 결과 텍스트
  unit text,              -- 단위 (예: 'mmHg', 'mg/dL')
  is_abnormal boolean,    -- 이상 여부
  created_at timestamp
)

-- 첨부 파일 (결과지 사진/PDF)
attachments (
  id uuid PK,
  record_id uuid FK,
  file_url text,
  file_type text,         -- 'image' | 'pdf'
  created_at timestamp
)

-- 사용자 ↔ 가족 연결
user_families (
  user_id uuid FK,        -- Supabase Auth user
  family_id uuid FK,
  role text               -- 'owner' | 'member'
)
```

## 검진 항목 내장 DB (item_definitions 테이블 or 상수)

### 항목 유형 2가지
- **표준항목**: 앱에 미리 내장. 설명, 정상범위, 단위 자동 표시
- **직접입력 항목**: 사용자가 결과지에서 처음 보는 항목을 직접 입력. 이름/수치만 저장 (설명 없음)

직접입력 항목도 저장 후 히스토리에 남으며, 다음 입력 시 "최근 입력 항목"으로 추천됨.

### 표준항목 카테고리 및 목록

> 암검진 등 묶음 검사는 **개별 검사 항목으로 분리 저장**한다.
> (예: 대장내시경과 분변잠혈은 주기가 다르므로 별도 항목)
> 카테고리는 분류 라벨일 뿐, 실제 항목은 개별 추적된다.

| 카테고리 | 항목 |
|---|---|
| 신체계측 | 신장, 체중, 체성분, 허리둘레, 1차혈압, 2차혈압(수축기/이완기), 맥박 |
| 혈액 | 혈색소(빈혈), 공복혈당, 총콜레스테롤, LDL, HDL, 중성지방, 비타민D |
| 간/신장 | AST, ALT, 감마GTP, 크레아티닌, 요단백, 요검사(채뇨) |
| 심장 | 심전도, 뇌심혈관검사 |
| 암검진(개별) | 위내시경, 위장조영촬영, 대장내시경, 분변잠혈검사, 저선량 폐CT |
| 여성검진 | 자궁경부세포검사(Pap), 인체유두종바이러스(HPV), 유방촬영술(MAMMO), 유방초음파, 3D유방초음파, 자궁/난소 초음파, 여성호르몬검사 |
| 초음파 | 경동맥초음파, 심장초음파, 갑상선초음파, 복부초음파 |
| 영상검사(MDCT/MRI) | 부비동 MDCT, 뇌 MDCT, 경추(목) MDCT, 요추(허리) MDCT, 복부 MDCT, 폐 MDCT, 골반 MDCT, 심장석회화 MDCT, 뇌 MRI |
| 안과 | 나안시력(좌/우), 교정시력(좌/우), 안압(좌/우), 안저검사 |
| 청력 | 청력검사 |
| 기타 | 흉부X선, 골밀도(DEXA), 인지기능, 우울증(정신문진), 노인신체기능검사 |

> 카테고리 분류는 **검사 방식/부위 기준**이되, 여성검진은 함께 받는 경우가 많아 별도 카테고리로 묶음.
> 위 목록은 실제 종합검진센터 항목표(강남 GIE 등) 기준으로 정리. 빈 항목은 사용자가 직접입력으로 추가 가능.

## 항목 조회 UI (카테고리 분류 방식)

### 기록 추가 시 (항목 선택)
- 카테고리 탭으로 표준항목 탐색 및 선택
- 없는 항목은 "직접입력"으로 추가 가능

### 히스토리 검색 시 (조회 전용)
- 직접 텍스트 입력 없이 저장된 항목만 선택 가능 (오타/중복 방지)
- **조회 화면에는 실제로 받은 항목만 표시** (안 받은 표준항목은 숨김)
- 카테고리는 받은 항목들을 분류하는 라벨로만 사용

```
[혈액]     혈당, 총콜레스테롤      ← 받은 항목만
[간/신장]  AST, ALT               ← 받은 항목만
[직접입력] 갑상선수치              ← 사용자가 추가한 항목
```

- 항목 선택 → 해당 구성원의 연도별 수치 리스트 표시

## 주요 기능 (1차 MVP)

### 1. 가족 그룹 & 구성원 관리
- 가족 그룹 생성
- 구성원 추가 (이름, 생년월일, 성별)
- 초대 링크로 가족 공유

### 2. 검진 기록 입력
- **건강검진**: 날짜/병원 입력 후 항목 여러 개 선택 및 수치 입력
- **단일검사**: 날짜/병원 + 항목 하나만 빠르게 입력
- 결과지 사진/PDF 첨부 (선택사항)

### 3. 항목 설명 & 정상범위
- 항목 선택 시 설명, 정상범위, 단위 표시
- "이 수치가 뭔가요?" 바로 확인 가능

### 4. 히스토리 조회
- **구성원별 보기**: 특정 구성원의 전체 검진 타임라인
- **항목별 조회**: 카테고리별로 분류된 항목 중 선택
  - 텍스트 직접 입력 없이 저장된 항목 목록에서만 선택 (오타/중복 방지)
  - 조회 화면에는 실제로 받은 항목만 표시 (카테고리는 분류용)
  - 선택 시 해당 구성원의 연도별 수치 리스트 표시
  - 마지막 검사일, 수치 추이 한눈에 확인

### 5. 가족 공유
- 초대 링크로 가족 구성원 초대
- 같은 가족 그룹 내 모든 기록 열람 가능
- Supabase Row Level Security로 타 가족 접근 차단

## 2차 개발 (추후)

- **결과지 OCR 자동 입력**: 검진 결과지 사진/PDF를 Claude API(이미지 인식)로 분석 → 항목·수치 자동 추출 및 입력. 공공 API 연계보다 범용적 (어떤 병원 결과지든 처리 가능)
- AI 채팅 상담 (Claude API 연동) - 내 수치 기반 질문/답변
- 수치 트렌드 차트 (연도별 변화 시각화)
- 영유아 예방접종 & 영유아검진 관리
- 검진 시기 알림 (푸시 알림)

> **참고**: 보건복지부 '나의건강기록' 앱의 공공데이터 자동연계(건강정보 고속도로 API)는 기관 심사·보안 인증이 필요해 개인 앱은 사용 불가. 대신 해당 앱에서 PDF를 내보내 OCR로 읽어들이는 방식으로 우회 가능.

## 모바일 앱 전환 전략 (추후)

웹앱(PWA)으로 먼저 개발·사용 후, 필요시 단계적으로 모바일 전환.
**백엔드(Supabase)와 로직(TypeScript)은 100% 재사용**되므로 갈아엎을 필요 없음.

```
1단계: 웹앱(PWA) 개발 & 사용  ← 현재 목표
2단계: Capacitor로 래핑 → iOS/안드로이드 앱스토어 등록 (코드 거의 그대로, 1~2일)
3단계: (선택) 인기 많아지면 React Native 재작성 검토 - 대개 불필요
```

건강검진 기록 앱은 목록/입력/검색 위주라 복잡한 애니메이션이 없어 Capacitor(웹뷰 래핑)로 충분. React Native는 오버스펙.

## 디자인 가이드 (확정)

깨끗하고 산뜻한 UI. 참고 톤: 똑닥/토스 같은 흰 배경 + 선명한 색 (열나요의 카드 구조는 참고하되 색은 차별화).

### 색상 (핵심)
- **배경: 순백(#FFFFFF)** — 크림/회색 배경 금지 (칙칙해 보임)
- **섹션 구분: 아주 옅은 회색(#F8F9FB)** — 카드/요약 블록 배경
- **메인 포인트: 선명한 파랑(#3B82F6)** — 채도 높게, 맑고 깨끗하게
- **연한 파랑(#EFF6FF)** — 선택 상태 배경
- 텍스트: 진한 먹색(#1A1D23), 보조는 회색(#8B92A0)
- **상태 뱃지: 정상=초록(#16A34A/#DCFCE7), 경계·이상=빨강(#DC2626/#FEE2E2)**
- 보조 강조색: 보라(#7C3AED, 단일검사 태그 등)

### 레이아웃
- **상단 인사형 헤더** + 알림 벨
- **가족 구성원 칩 (가로 스크롤)**: 이모지(밝은 피부톤 👨🏻👩🏻👦🏻) + 이름
  - 선택 시: 연파랑 배경 + 살짝 확대(scale 1.05), **그림자 없음**
  - 비선택: 옅은 회색 배경 + 이모지 흐리게(opacity 0.55) + 회색 글씨
- **다음 검진 알림 카드**: 살짝 연한 진파랑(#5B9BF5) 배경 + 흰 글씨 (또렷하게 — 단, 다른 진파랑 요소와 동시에 두지 않아 시선 충돌 방지)
- **빠른 메뉴**: 4칸 그리드, 기능별 파스텔 컬러 아이콘(연파랑/연초록/연빨강/연노랑 배경)
- **최근 검진 카드**: 옅은 회색 배경, 항목명+상태뱃지 / 큰 수치+단위, 항목별 정상범위 표시
- **검진 타임라인**: 세로선 + 점(dot)으로 연결, 병원명+유형태그+날짜

### 타이포그래피
- 큰 숫자/핵심 지표는 굵고 크게
- 제목 굵게, 본문 보통, 보조설명 작고 연하게 — 위계 명확히

### 구성원 프로필
- 이모지 선택(기본, 밝은 피부톤) + 사진 업로드(선택)

### 구현 메모
- Tailwind CSS, 모바일 우선(375px), 둥근 모서리(rounded-2xl/3xl) 일관 적용
- 진한 색은 한 화면에 한 곳만 강하게 — 시선이 분산되지 않게
