# 가족 건강검진 관리 앱 - 프로젝트 설계

## 프로젝트 개요
가족 구성원의 건강검진 결과를 저장·관리하는 개인용 웹앱.
매년 검진 시 이전 기록을 쉽게 확인하고, 병원 방문 시 검사 이력을 즉시 조회할 수 있게 한다.

---

## 기술 스택

| 영역 | 기술 |
|---|---|
| Frontend | Next.js 14 (App Router) + TypeScript + Tailwind CSS |
| Backend/DB | Supabase (PostgreSQL + Auth + Storage) |
| 배포 | Vercel (프론트) + Supabase (백엔드) |
| 플랫폼 | 웹앱 (모바일 브라우저 최적화 PWA) |

## 데이터 모델

```sql
-- 가족 그룹
families (
  id uuid PK,
  name text,              -- "홍길동 가족"
  invite_code text,       -- 초대 링크용 코드
  created_at timestamp
)

-- 가족 구성원
members (
  id uuid PK,
  family_id uuid FK,
  name text,
  birth_date date,
  gender text,            -- 'M' | 'F'
  created_at timestamp
)

-- 검진 기록 (건강검진 or 단일검사 모두)
checkup_records (
  id uuid PK,
  member_id uuid FK,
  type text,              -- 'checkup' (건강검진) | 'single' (단일검사)
  record_date date,
  hospital text,
  notes text,
  created_at timestamp
)

-- 항목별 결과 수치
checkup_items (
  id uuid PK,
  record_id uuid FK,
  item_code text,         -- 표준 항목 코드 (예: 'blood_pressure_sys')
  item_name text,         -- 표시 이름 (예: '수축기혈압')
  value text,             -- 수치 or 결과 텍스트
  unit text,              -- 단위 (예: 'mmHg', 'mg/dL')
  is_abnormal boolean,    -- 이상 여부
  created_at timestamp
)

-- 첨부 파일 (결과지 사진/PDF)
attachments (
  id uuid PK,
  record_id uuid FK,
  file_url text,
  file_type text,         -- 'image' | 'pdf'
  created_at timestamp
)

-- 사용자 ↔ 가족 연결
user_families (
  user_id uuid FK,        -- Supabase Auth user
  family_id uuid FK,
  role text               -- 'owner' | 'member'
)
```

## 검진 항목 내장 DB (item_definitions 테이블 or 상수)

### 항목 유형 2가지
- **표준항목**: 앱에 미리 내장. 설명, 정상범위, 단위 자동 표시
- **직접입력 항목**: 사용자가 결과지에서 처음 보는 항목을 직접 입력. 이름/수치만 저장 (설명 없음)

직접입력 항목도 저장 후 히스토리에 남으며, 다음 입력 시 "최근 입력 항목"으로 추천됨.

### 표준항목 카테고리 및 목록

> 암검진 등 묶음 검사는 **개별 검사 항목으로 분리 저장**한다.
> (예: 대장내시경과 분변잠혈은 주기가 다르므로 별도 항목)
> 카테고리는 분류 라벨일 뿐, 실제 항목은 개별 추적된다.

| 카테고리 | 항목 |
|---|---|
| 신체계측 | 신장, 체중, 체성분, 허리둘레, 1차혈압, 2차혈압(수축기/이완기), 맥박 |
| 혈액 | 혈색소(빈혈), 공복혈당, 총콜레스테롤, LDL, HDL, 중성지방, 비타민D |
| 간/신장 | AST, ALT, 감마GTP, 크레아티닌, 요단백, 요검사(채뇨) |
| 심장 | 심전도, 뇌심혈관검사 |
| 암검진(개별) | 위내시경, 위장조영촬영, 대장내시경, 분변잠혈검사, 저선량 폐CT |
| 여성검진 | 자궁경부세포검사(Pap), 인체유두종바이러스(HPV), 유방촬영술(MAMMO), 유방초음파, 3D유방초음파, 자궁/난소 초음파, 여성호르몬검사 |
| 초음파 | 경동맥초음파, 심장초음파, 갑상선초음파, 복부초음파 |
| 영상검사(MDCT/MRI) | 부비동 MDCT, 뇌 MDCT, 경추(목) MDCT, 요추(허리) MDCT, 복부 MDCT, 폐 MDCT, 골반 MDCT, 심장석회화 MDCT, 뇌 MRI |
| 안과 | 나안시력(좌/우), 교정시력(좌/우), 안압(좌/우), 안저검사 |
| 청력 | 청력검사 |
| 기타 | 흉부X선, 골밀도(DEXA), 인지기능, 우울증(정신문진), 노인신체기능검사 |

> 카테고리 분류는 **검사 방식/부위 기준**이되, 여성검진은 함께 받는 경우가 많아 별도 카테고리로 묶음.
> 위 목록은 실제 종합검진센터 항목표(강남 GIE 등) 기준으로 정리. 빈 항목은 사용자가 직접입력으로 추가 가능.

## 항목 조회 UI (카테고리 분류 방식)

### 기록 추가 시 (항목 선택)
- 카테고리 탭으로 표준항목 탐색 및 선택
- 없는 항목은 "직접입력"으로 추가 가능

### 히스토리 검색 시 (조회 전용)
- 직접 텍스트 입력 없이 저장된 항목만 선택 가능 (오타/중복 방지)
- **조회 화면에는 실제로 받은 항목만 표시** (안 받은 표준항목은 숨김)
- 카테고리는 받은 항목들을 분류하는 라벨로만 사용

```
[혈액]     혈당, 총콜레스테롤      ← 받은 항목만
[간/신장]  AST, ALT               ← 받은 항목만
[직접입력] 갑상선수치              ← 사용자가 추가한 항목
```

- 항목 선택 → 해당 구성원의 연도별 수치 리스트 표시

## 주요 기능 (1차 MVP)

### 1. 가족 그룹 & 구성원 관리
- 가족 그룹 생성
- 구성원 추가 (이름, 생년월일, 성별)
- 초대 링크로 가족 공유

### 2. 검진 기록 입력
- **건강검진**: 날짜/병원 입력 후 항목 여러 개 선택 및 수치 입력
- **단일검사**: 날짜/병원 + 항목 하나만 빠르게 입력
- 결과지 사진/PDF 첨부 (선택사항)

### 3. 항목 설명 & 정상범위
- 항목 선택 시 설명, 정상범위, 단위 표시
- "이 수치가 뭔가요?" 바로 확인 가능
- → **6단계에서 "검진 항목 사전(/guide)" 으로 구현됨.** 아래 4번 참고.

### 4. 히스토리 조회 & 검진 항목 사전 (/guide)
> ⚠️ **설계 변경(2026-06-27):** 원래 "항목별 조회 = 받은 항목의 연도별 수치 리스트"였으나,
> 사용자의 실제 니즈("건강검진 신청할 때 뭐가뭔지, 작년에 뭘 받았는지, 내 나이엔 뭘 받아야 하는지 모르겠다")에 맞춰
> **"검진 항목 사전"** 으로 재정의함. 단순 수치 나열이 아니라 설명·권장주기·내 이력을 한 화면에 통합.
> 자세한 배경은 문서 맨 아래 "설계 변경 이력" 참고.

- **구성원별 타임라인**: 홈 화면(HomeView)에서 선택 구성원의 전체 검진 타임라인 표시. 기록 클릭 → 기록 상세(/records/[id])
- **검진 항목 사전(/guide)**: 표준항목(item_definitions ~70개)을 카테고리/검색으로 탐색하는 조회 화면
  - **목록**: 검색 + 카테고리 탭 + 구성원 칩. 각 항목에 그 구성원 기준 **상태 뱃지** 표시
    - `받을 때 됨`(노랑): 권장 주기가 지났거나, 권장 대상인데 받은 적 없음
    - `YYYY.MM 받음`(초록/파랑): 받은 기록 있음
    - `받은 적 없음`(회색): 권장 대상 아니고 기록도 없음
  - **상세**: 항목 하나를 누르면 세 블록 — ①설명+정상범위 ②연령·성별 권장주기 ③그 구성원의 검사 이력(시간순, 각 이력은 기록 상세로 링크)
  - 상태 뱃지/권장주기는 구성원의 나이·성별로 판정 (lib/screeningGuide.ts)
  - ③의 검사 이력이 곧 "항목별 연도 추이"라서, 2차 개발의 "수치 트렌드 차트"는 이 이력 리스트로 일부 흡수됨 (차트 시각화는 추후)

### 5. 가족 공유
- 초대 링크로 가족 구성원 초대
- 같은 가족 그룹 내 모든 기록 열람 가능
- Supabase Row Level Security로 타 가족 접근 차단

## 2차 개발 (추후)

- **결과지 OCR 자동 입력**: 검진 결과지 사진/PDF를 Claude API(이미지 인식)로 분석 → 항목·수치 자동 추출 및 입력. 공공 API 연계보다 범용적 (어떤 병원 결과지든 처리 가능)
- AI 채팅 상담 (Claude API 연동) - 내 수치 기반 질문/답변
- 수치 트렌드 차트 (연도별 변화 **시각화**) — 데이터 나열은 이미 항목 사전 상세 ③이력으로 제공됨. 추후 차트(그래프) 형태만 추가
- 영유아 예방접종 & 영유아검진 관리
- 검진 시기 알림 (푸시 알림) — 항목 사전의 "받을 때 됨" 판정(screeningGuide) 로직을 그대로 활용 가능

> **참고**: 보건복지부 '나의건강기록' 앱의 공공데이터 자동연계(건강정보 고속도로 API)는 기관 심사·보안 인증이 필요해 개인 앱은 사용 불가. 대신 해당 앱에서 PDF를 내보내 OCR로 읽어들이는 방식으로 우회 가능.

## 화면 구성 (페이지)

```
/ (홈) — HomeView
  ├── 인사형 헤더 + 마이페이지 버튼
  ├── 가족 구성원 칩 (가로 스크롤, 선택 유지)
  ├── 빠른 메뉴 3칸 (기록 추가 / 항목 사전 / 결과지)
  └── 선택 구성원의 검진 기록 타임라인
      ※ 과거엔 "최근 검진" 요약 카드가 있었으나 제거됨 (설계 변경 이력 참고)

/members/[id] (구성원 상세)
  └── 검진 기록 타임라인

/members/new (구성원 추가) — MemberForm

/records/new (기록 추가) — RecordForm (mode="create")
  ├── 유형 선택 (건강검진 / 단일검사)
  ├── 구성원은 진입 시 고정(헤더에 표시, 입력칸 없음)
  └── 항목 입력 폼 (항목은 기본 순서 sort_order로 자동 정렬)

/records/[id] (기록 상세) — RecordView(조회) ↔ RecordForm(편집)
  ├── 진입 시 읽기 전용 조회(RecordView)
  └── 우상단 "편집" 버튼 → RecordForm(mode="edit")로 전환, 저장/취소 시 조회로 복귀

/guide (검진 항목 사전) — GuideView(목록) ↔ GuideDetail(상세)
  ├── 검색 + 카테고리 + 구성원 칩 + 상태 뱃지
  └── 항목 클릭 → 상세(설명/권장주기/내 이력). 목록↔상세는 클라 state 토글
      (홈의 "항목 검색" 빠른메뉴 = 이 페이지. /search 가 아님)

/mypage (마이페이지)
  └── 이메일 / 로그아웃 / 가족 이름 변경(FamilyNameEditor)

/family (가족 관리) — ※ 8단계에서 구현 예정
  └── 초대 링크 생성 / 구성원 관리
```

## 모바일 앱 전환 전략 (추후)

웹앱(PWA)으로 먼저 개발·사용 후, 필요시 단계적으로 모바일 전환.
**백엔드(Supabase)와 로직(TypeScript)은 100% 재사용**되므로 갈아엎을 필요 없음.

```
1단계: 웹앱(PWA) 개발 & 사용  ← 현재 목표
2단계: Capacitor로 래핑 → iOS/안드로이드 앱스토어 등록 (코드 거의 그대로, 1~2일)
3단계: (선택) 인기 많아지면 React Native 재작성 검토 - 대개 불필요
```

건강검진 기록 앱은 목록/입력/검색 위주라 복잡한 애니메이션이 없어 Capacitor(웹뷰 래핑)로 충분. React Native는 오버스펙.

## 디자인 가이드 (확정)

깨끗하고 산뜻한 UI. 참고 톤: 똑닥/토스 같은 흰 배경 + 선명한 색 (열나요의 카드 구조는 참고하되 색은 차별화).

### 색상 (핵심)
- **배경: 순백(#FFFFFF)** — 크림/회색 배경 금지 (칙칙해 보임)
- **섹션 구분: 아주 옅은 회색(#F8F9FB)** — 카드/요약 블록 배경
- **메인 포인트: 선명한 파랑(#3B82F6)** — 채도 높게, 맑고 깨끗하게
- **연한 파랑(#EFF6FF)** — 선택 상태 배경
- 텍스트: 진한 먹색(#1A1D23), 보조는 회색(#8B92A0)
- **상태 뱃지: 정상=초록(#16A34A/#DCFCE7), 경계·이상=빨강(#DC2626/#FEE2E2)**
- 보조 강조색: 보라(#7C3AED, 단일검사 태그 등)

### 레이아웃
- **상단 인사형 헤더** + 알림 벨
- **가족 구성원 칩 (가로 스크롤)**: 이모지(밝은 피부톤 👨🏻👩🏻👦🏻) + 이름
  - 선택 시: 연파랑 배경 + 살짝 확대(scale 1.05), **그림자 없음**
  - 비선택: 옅은 회색 배경 + 이모지 흐리게(opacity 0.55) + 회색 글씨
- **다음 검진 알림 카드**: (디자인 시안상 아이디어. 현재 홈에는 미구현 — 검진 주기 가정의 정확성 문제로 보류. 항목 사전의 "받을 때 됨" 판정으로 대체 검토)
- **빠른 메뉴**: 3칸 그리드(기록 추가 / 항목 사전 / 결과지), 기능별 파스텔 컬러 아이콘. (원래 4칸이었으나 "수치 추이"를 항목 사전에 통합하며 3칸으로 축소 — 변경 이력 참고)
- ~~**최근 검진 카드**~~: **제거됨** (2026-06-27). 4개 항목만 미리보기하는 카드가 바로 아래 타임라인과 정보 중복 + 불필요한 탭 유발이라 삭제. 자세한 이유는 변경 이력 참고.
- **검진 타임라인**: 세로선 + 점(dot)으로 연결, 병원명+유형태그+날짜. 기록 없으면 빈 상태 안내 카드

### 타이포그래피

## 설계 변경 이력

> 이 문서(CLAUDE.md)는 최초 설계안이다. 개발하며 의도가 바뀐 결정은 여기에 **왜 바꿨는지까지** 기록한다.
> (진행 상황·구현 디테일 로그는 별도 메모리 파일에 둠. 여기는 "설계 의사결정"만.)

### 2026-06-27 — 6단계 재정의 외 홈/기록 UX 변경

**1. "항목별 조회"를 "검진 항목 사전(/guide)"으로 재정의**
- 변경 전 설계: 받은 항목을 골라 연도별 수치 리스트만 보여주는 조회 화면.
- 바꾼 이유: 사용자의 진짜 불편은 "수치 나열"이 아니라 **"건강검진 신청할 때 항목마다 뭐가뭔지 모르겠고, 작년·재작년에 뭘 받았는지도 모르겠고, 내 나이엔 뭘 받아야 하는지 모르겠다"** 였음.
- 바꾼 내용: 단순 조회 → **설명 + 권장주기 + 내 이력**을 한 화면에 합친 "사전". 받은 적 없는 표준항목도 (지식 조회 목적으로) 목록에 나옴. 단, 직접입력(item_code=null) 항목은 사전에 안 나옴(표준항목만).
- 권장주기 데이터(lib/screeningGuide.ts): 국가 일반건강검진 / 6대 암검진 / 골다공증 기준. **모든 항목에 다 넣지 않음** — 국가검진·흔한 항목 위주로만 채우고 나머진 "권장 정보 없음"으로. (전 항목 권장주기를 정의하는 건 근거 부족 + 알림 피로 우려)

**2. "수치 추이(2차 개발)"를 항목 사전 상세 ③이력으로 흡수**
- 항목별 시간순 이력 = 사실상 수치 추이와 같은 데이터라 중복. 사전 상세에서 이력 리스트로 제공.
- 단, **차트(그래프) 시각화**는 여전히 2차 개발 과제로 남김. 지금은 리스트 형태.

**3. 홈 "최근 검진" 요약 카드 제거**
- 변경 전: 최신 검진의 항목 4개를 미리보기하는 카드가 타임라인 위에 있었음.
- 바꾼 이유: ①바로 아래 타임라인과 정보 중복 ②4개만 보여줘 정보가 어중간 ③단일검사는 항목명 없이 수치만, 건강검진은 "이상 N개"만 보여 **불필요한 탭을 유발**(눌러봐도 할 게 별로 없음).
- 결과: 홈 = 구성원 칩 → 빠른 메뉴 → 타임라인으로 단순화. 빈 상태 안내는 타임라인 자리에 통합.

**4. 검진 기록 상세를 "조회 우선"으로 분리 (RecordView ↔ RecordForm)**
- 변경 전: /records/[id] 진입 시 곧장 편집 폼(RecordForm)이 떴음.
- 바꾼 이유: 보기만 하려는데 편집 화면이 떠서 실수 수정·이탈 경고 등 마찰. 조회와 편집은 다른 의도.
- 결과: 진입 시 읽기 전용 RecordView, 우상단 "편집" 버튼 눌러야 RecordForm(mode=edit). 저장/취소 시 조회로 복귀(onCancelEdit). 삭제 버튼은 조회 화면 하단.

**5. 검진 기록 항목 자동 정렬**
- 입력한 순서가 아니라 **기본 순서(item_definitions.sort_order, 자주 받는 순)** 로 자동 정렬. 직접입력 항목은 맨 뒤(추가 순서 유지). 표시·저장 순서 모두 일관.

**6. 빠른 메뉴 4칸 → 3칸**
- "수치 추이"를 항목 사전에 통합하며 제거. 현재 [기록 추가 / 항목 사전 / 결과지]. (결과지는 7단계까지 placeholder)

**검토했으나 안 한 것 (재논의 방지용 기록)**
- **다음 검진 안내 카드(홈)**: 마지막 검진일 + 권장주기로 "다음 검진까지 N개월" 표시안. 검진 주기를 앱이 일률 가정하면(예: 누구나 2년) 직장인 1년 주기 등과 안 맞아 **틀린 정보**를 줄 수 있어 보류. 항목 사전의 항목별 "받을 때 됨"으로 대체.
- **병원 지도 검색(기록 입력)**: 카카오/네이버 지도 API 연동은 앱키·쿼터·도메인·CORS 등 운영 공수가 커서 안 함. 대신 기존 입력 병원명 자동완성 수준으로 충분(구성원별 최근 병원은 이미 일부 반영).
- **테스트 코드**: 개인용 PWA·UI 중심이라 ROI 낮아 작성 안 함(사용자 합의).
